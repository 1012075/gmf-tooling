Annotated Models

This package contains classes that derive graphical definition, tooling definition and mapping GMF models from specially annotated domain (ecore) model. Using these model setup classes user may easily define data for GMF unit tests. Generator model could be obtained by transforming mapping model. Essentially this is a simple alternative to GMF definition models that is specially designed to provide unit tests data.

Code that infers other models from domain model searches for annotations with 'gmf' source and uses details entries of annotations to produce related model elements. Here is a list of annotation details entry names supported in domain model:

  • canvas - annotated EClass will be used as diagram canvas element
  • node - node mapping with creation tool and diagram node with figure will be created for the annotated EClass
  • compartment - compartment mapping with diagram element is created; value of details entry specifies compartment name; compartment is defined within node mapping of the annotated EClass
  • in - value of details entry specifies name of the compartment defined in parent EClass
  • link - link mapping with creation tool and diagram connection with figure will be created for the annotated EClass or EReference
  • containment - name of the containment feature specified as EClass name plus dot plus feature name such as 'Library.writers'
  • source - name of the source feature for the link mapping
  • target - name of the target feature for the link mapping
  • label - label mapping will be defined for the annotated EAttribute
